TP ORM JPA - Les animaux

Projet initial

Vous devez télécharger le projet initial animaux-orm.zip.

Ce projet contient les classes des entités ainsi que la classe com.animoz.Application fournissant un exemple d’initialisation de JPA. Vous trouverez le fichier src/main/sql/database.sql qui vous permettra de créer le schéma de base de données dans MySQL ou MariaDB. Vous trouverez également le fichier src/main/resources/META-INF/persistence.xml vous permettant de configurer l’accès à la base de données (pilote JDBC, URL d’accès à la base de données, nom d’utilisateur et mot de passe).

Programmes à implémenter

Pour chaque fonctionnalité ci-dessous, créez une classe Java sur le modèle de la classe com.animoz.Application :

  • Afficher sur la console la liste des espèces.

  • Afficher sur la console la liste des animaux (nom et description).

  • Afficher sur la console la liste des animaux carnivores

  • Afficher sur la console la liste des animaux d’une espèce (codée en dur). Pour cela, il faut introduire une relation entre les classes Espece et Animal.

TP Spring Framework - Animoz le site Web

On souhaite créer un site en ligne pour permettre de gérer un zoo (ou un refuge pour animaux sauvages… comme vous préférez) appelé Animoz.

Les locaux d’Animoz comprennent un ensemble d’enclos qui portent un numéro. Chaque enclos peut accueillir plusieurs populations d’animaux. Les animaux sont classés selon leur espèce et leur régime alimentaire (carnivore ou herbivore).

Projet initial

Vous devez télécharger le projet initial animoz.zip.

Ce projet contient la base du site Web. Vous trouverez le fichier src/main/sql/database.sql qui vous permettra de créer le schéma de base de données dans MySQL ou MariaDB. Vous trouverez également le fichier src/main/resources/jdbc.properties vous permettant de configurer l’accès à la base de données (pilote JDBC, URL d’accès à la base de données, nom d’utilisateur et mot de passe).

Fonctionnalités à améliorer

À partir du projet fourni, on souhaite effectuer les améliorations suivantes :

  • On souhaite pouvoir supprimer une espèce (pour que l’opération soit possible, il ne doit pas exister d’animal de cette espèce dans la base de données)

  • On souhaite vérifier au moment de l’ajout d’une nouvelle espèce que le nom de l’espèce est bien renseigné.

  • On souhaite pouvoir ajouter un nouvel animal en fournissant ses caractéristiques : nom, origine, espèce, description, régime alimentaire.

  • On souhaite pouvoir modifier la fiche d’un animal existant

Nouvelles fonctionnalités

  • On souhaite déclarer une population d’animaux en indiquant le nombre d’individus et le numéro de l’enclos. Des populations d’animaux différents peuvent cohabiter dans le même enclos.

  • On veut s’assurer du respect des certaines règles lors de la déclaration d’une population. S’il s’agit d’une population d’animaux carnivores, alors elle ne peut être placée qu’avec des populations de la même espèce. S’il s’agit d’une population d’animaux herbivores, alors elle peut être placée avec n’importe quelles populations d’animaux herbivores.
